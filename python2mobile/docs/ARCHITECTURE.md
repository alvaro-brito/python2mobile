Runtime Engine\n\n**Responsibility:** Execute Python code and render component trees\n\n#### Components:\n\n**runtime.py**\n- `Runtime` - Safe Python execution sandbox\n- `Render` - Main entry point for rendering\n\n**render_engine.py**\n- `RenderEngine` - Converts component trees to HTML\n- Tailwind CSS class mapping\n- Mobile frame generation\n\n**ast_walker.py**\n- `ASTWalker` - Analyzes component tree structure\n- `CodeExtractor` - Extracts Python code information\n\n**hot_reload.py** (Phase 4)\n- File watcher for development\n- Auto-reload on changes\n\n**event_bridge.py** (Phase 4)\n- JavaScript to Python event bridge\n- WebSocket communication\n\n### 2. p2m.ui - Component Library\n\n**Responsibility:** Provide declarative UI components\n\n#### Component Hierarchy:\n\n```\nComponent (base class)\nв”ңв”Җв”Җ Container\nв”ңв”Җв”Җ Text\nв”ңв”Җв”Җ Button\nв”ңв”Җв”Җ Input\nв”ңв”Җв”Җ Image\nв”ңв”Җв”Җ List\nв”ңв”Җв”Җ Card\nв”ңв”Җв”Җ Badge\nв”ңв”Җв”Җ Icon\nв”ңв”Җв”Җ Modal\nв”ңв”Җв”Җ ScrollView\nв”ңв”Җв”Җ Row\nв”ңв”Җв”Җ Column\nв”ңв”Җв”Җ Navigator\nв”ңв”Җв”Җ Screen\nв””в”Җв”Җ ... more components\n```\n\n#### Component Structure:\n\n```python\n@dataclass\nclass Component:\n    component_type: str\n    props: Dict[str, Any]\n    children: List[Component]\n    \n    def add(self, child: Component) -> Component\n    def build(self) -> Dict[str, Any]\n```\n\n### 3. p2m.llm - LLM Integration\n\n**Responsibility:** Multi-provider LLM support\n\n#### Architecture:\n\n```\nLLMProvider (abstract base)\nв”ңв”Җв”Җ OpenAIProvider\nв”ңв”Җв”Җ AnthropicProvider\nв”ңв”Җв”Җ OllamaProvider\nв””в”Җв”Җ CompatibleProvider\n\nLLMFactory (factory pattern)\nв””в”Җв”Җ Creates appropriate provider based on config\n\nLLMResponse (data class)\nв””в”Җв”Җ Wraps LLM responses with metadata\n```\n\n#### Provider Features:\n\n| Provider | Features |\n|----------|----------|\n| OpenAI | gpt-4o, gpt-4-turbo, streaming |\n| Anthropic | Claude 3, streaming, vision |\n| Ollama | Local models, no API key |\n| Compatible | Custom base_url, headers |\n\n### 4. p2m.build - Code Generation\n\n**Responsibility:** Generate platform-specific code\n\n#### Components:\n\n**generator.py**\n- `CodeGenerator` - Main code generation engine\n- Loads project files\n- Builds LLM prompts\n- Generates React Native or Flutter code\n\n**react_native.py** (Phase 5)\n- React Native templates\n- Expo configuration\n\n**flutter.py** (Phase 5)\n- Flutter templates\n- Material Design mapping\n\n**cache.py** (Phase 5)\n- Build caching system\n- SHA256 hash-based invalidation\n\n### 5. p2m.devserver - Development Server\n\n**Responsibility:** Provide development environment\n\n#### Components:\n\n**server.py**\n- FastAPI-based HTTP server\n- WebSocket support\n- Mobile frame rendering\n- Tailwind CDN integration\n\n**templates.py** (Phase 4)\n- Jinja2 HTML templates\n- Mobile viewport emulation\n\n**websocket_handler.py** (Phase 4)\n- Event bridge implementation\n- Hot reload notifications\n\n### 6. p2m.config - Configuration Management\n\n**Responsibility:** Handle project configuration\n\n#### Configuration Structure:\n\n```\nConfig\nв”ңв”Җв”Җ ProjectConfig\nв”Ӯ   в”ңв”Җв”Җ name\nв”Ӯ   в”ңв”Җв”Җ version\nв”Ӯ   в””в”Җв”Җ entry\nв”ңв”Җв”Җ BuildConfig\nв”Ӯ   в”ңв”Җв”Җ target\nв”Ӯ   в”ңв”Җв”Җ generator\nв”Ӯ   в”ңв”Җв”Җ llm_provider\nв”Ӯ   в”ңв”Җв”Җ llm_model\nв”Ӯ   в”ңв”Җв”Җ output_dir\nв”Ӯ   в””в”Җв”Җ cache\nв”ңв”Җв”Җ DevServerConfig\nв”Ӯ   в”ңв”Җв”Җ port\nв”Ӯ   в”ңв”Җв”Җ hot_reload\nв”Ӯ   в””в”Җв”Җ mobile_frame\nв”ңв”Җв”Җ StyleConfig\nв”Ӯ   в””в”Җв”Җ system\nв””в”Җв”Җ LLMConfig\n    в”ңв”Җв”Җ provider\n    в”ңв”Җв”Җ api_key\n    в”ңв”Җв”Җ model\n    в”ңв”Җв”Җ base_url\n    в””в”Җв”Җ x_api_key\n```\n\n### 7. p2m.cli - Command Line Interface\n\n**Responsibility:** Provide CLI commands\n\n#### Commands:\n\n```bash\np2m run           # Run development server\np2m build         # Build for production\np2m new <name>    # Create new project\np2m info          # Show project info\n```\n\n## Data Flow\n\n### Development Flow (p2m run)\n\n```\n1. User executes: p2m run\n   вҶ“\n2. CLI loads p2m.toml configuration\n   вҶ“\n3. Runtime imports main.py module\n   вҶ“\n4. Runtime executes create_view() function\n   вҶ“\n5. Component tree is built and returned\n   вҶ“\n6. RenderEngine converts tree to HTML\n   вҶ“\n7. DevServer starts on localhost:3000\n   вҶ“\n8. Browser displays app with mobile frame\n   вҶ“\n9. Watchdog monitors .py files\n   вҶ“\n10. On file change вҶ’ hot reload\n```\n\n### Build Flow (p2m build)\n\n```\n1. User executes: p2m build --target android\n   вҶ“\n2. CLI loads p2m.toml configuration\n   вҶ“\n3. CodeGenerator loads all .py files\n   вҶ“\n4. AST Walker extracts structure\n   вҶ“\n5. LLM prompt is built from code\n   вҶ“\n6. LLM generates platform code\n   вҶ“\n7. Generated code is validated\n   вҶ“\n8. Code is written to /build/android/\n   вҶ“\n9. Build cache is updated\n   вҶ“\n10. Build complete\n```\n\n## Component Rendering Pipeline\n\n```\nPython Code\n    вҶ“\nComponent Tree (Dict)\n    вҶ“\nRenderEngine\n    в”ңв”Җ Extract props\n    в”ңв”Җ Map to HTML tags\n    в”ңв”Җ Apply Tailwind classes\n    в””в”Җ Render recursively\n    вҶ“\nHTML String\n    вҶ“\nBrowser\n    в”ңв”Җ Parse HTML\n    в”ңв”Җ Apply Tailwind CSS\n    в”ңв”Җ Render mobile frame\n    в””в”Җ Display app\n```\n\n## Type System\n\n### Component Tree Structure\n\n```python\nComponentTree = Dict[str, Any]\n\n{\n    \"type\": \"Container\",\n    \"props\": {\n        \"class\": \"bg-white p-4\",\n        \"direction\": \"column\",\n    },\n    \"children\": [\n        {\n            \"type\": \"Text\",\n            \"props\": {\n                \"value\": \"Hello\",\n                \"class\": \"text-lg\"\n            },\n            \"children\": []\n        }\n    ]\n}\n```\n\n### LLM Response Structure\n\n```python\nLLMResponse = {\n    \"content\": str,      #
